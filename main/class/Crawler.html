Class: Crawler

Defined in: src/scraper/Crawler.coffee

Overview

The Crawler coordinates execution of submitted CrawlRequest by applying all Extensions matching the request's current status.

All functionality for request handling, such as filtering, queueing, streaming, storing, logging etc. is implemented as Extensions to ExtensionPoints.

The crawler defines an ExtensionPoint for each distinct value of RequestStatus. Each ExtensionPoint wraps the processing steps carried out when the request status changes to a new value. The status transitions implicitly define a request flow illustrated in the diagram below.

 .-------------.
 |   INITIAL   |
 |-------------|
 | Unprocessed |
 |             |
 '-------------'   \
        |           \
        |            \
        |             v
        v             .--------------------.
 .-------------.      |  ERROR | CANCELED  |      .-----------.
 |  SPOOLED    |      |--------------------|      | COMPLETED |
 |-------------|  --->| - Error            |      |-----------|
 | Waiting for |      | - Duplicate        |      | Done!     |
 | free slot   |      | - Blacklisted etc. |      |           |
 '-------------'      '--------------------'      '-----------'
        |             ^         ^          ^            ^
        |            /          |           \           |
        |           /           |            \          |
        v          /                          \         |
 .-------------.         .-------------.          .-----------.
 |    READY    |         |  FETCHING   |          |  FETCHED  |
 |-------------|         |-------------|          |-----------|
 | Ready for   |-------->| Request     |--------->| Content   |
 | fetching    |         | streaming   |          | received  |
 '-------------'         '-------------'          '-----------'
