Class: CrawlRequest

Defined in: src/kermit/CrawlRequest.coffee

Overview

The crawl request is the central object of processing. It is not to be confused with an Http(s) request (which might be created in the lifespan of a crawl request). Each crawl request has a lifecycle determined by the state diagram as defined by the Crawler and its ExtensionPoints. During its lifecycle the request is enriched with listeners and properties by the Extensions that take care of its processing. Any information necessary for request processing is usually to the request in order to centralize state. Any property added to its internal state {CrawlRequest#state} will be persistent after the next status transition.
