Class: RequestStatus

Defined in: src/scraper/CrawlRequest.coffee

Overview

At any time, each request has a status value equal to one of the values defined by this class. Any request starts with status INITIAL (set in constructor). From status INITIAL it transitions forward while being processed by the {Extensions}s that handle requests of that particular status. A full state diagram of the status transitions is found in the documentation of the Crawler

Property Summary

(String) INITIAL
The first status of any request
(String) SPOOLED
Spooled requests are waiting in the queuing system QueueManager for further processing
(String) READY
Request is ready to be fetched by the {Streamer}
(String) FETCHING
Http(s) call is made and response is being streamed
(String) FETCHED
All data has been received and the response is ready for further processing
(String) COMPLETE

Response processing is finished. This is the terminal status of a successfully processed request

(String) ERROR
ExtensionPoints will set this status if an exception occurs during execution of an Extension
(String) CANCELED

Any extension might cancel a request. Canceled requests are not elligible for further processing and will be cleaned up, see Cleanup

(Array<String>) ALL
Collection of all defined status'
